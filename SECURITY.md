# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in RaisinDB, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **security@solutas.ch**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 7 days
- **Fix**: Dependent on severity, typically within 30 days

## Supported Versions

Security updates are provided for the latest release only.

## Scope

This policy applies to the RaisinDB server, client libraries, and official tooling in this repository.
